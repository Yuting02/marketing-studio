# 出海营销内容工作台 — PRD v1（MVP）

> 状态：已锁定（MVP 范围）｜维护：Yuting
> 项目的核心 harness 文档。开发中如有范围调整，**先改这里，再让 AI 照着做**。

## 1. 一句话
输入产品信息，自动生成 Meta（Facebook/Instagram）多语种广告文案，并对每条文案做合规质检与修改建议。

## 2. 目标用户
做出海投放的营销 / 运营人员，需要快速产出并初筛多语种广告文案。

## 3. 核心流程（两步 LLM 工作流）
1. 用户输入：产品名 + 卖点（2–3 句）+ 目标语种
2. **生成节点**：LLM 输出结构化广告文案（每语种 3 组变体）
3. **质检节点**：LLM 按规则对每组文案打分 → 通过 / 风险 + 命中规则 + 修改建议
4. 界面卡片展示文案 + 质检结果，可一键复制

## 4. MVP 范围（已锁定）
- **平台**：Meta（Facebook / Instagram）单一平台
- **语种**：英语（en）+ 法语（fr）
- **每次生成**：每语种 3 组变体
- **广告字段（每组）**：
  - 主文案 `primaryText`（推荐 ≤125 字符）
  - 标题 `headline`（推荐 ≤40 字符）
  - 描述 `description`（推荐 ≤30 字符）
  - 行动号召 `cta`（枚举：`SHOP_NOW` / `LEARN_MORE` / `SIGN_UP` / `GET_OFFER`）
- **质检规则（内置 4 条）**：
  1. **字段长度**：超出推荐上限 → 风险
  2. **绝对化 / 夸大违禁词**：best、#1、guaranteed、no.1 等 → 风险
  3. **敏感 / 违禁内容**：歧视、医疗承诺、虚假宣称等 → 风险
  4. **语言地道度**：0–100 评分，低于阈值（如 70）→ 风险
- **输出**：卡片展示 + 一键复制

> 字符上限和违禁词都做成**可配置**（写在一处常量文件里），方便讲“规则即配置 / 误杀漏放调参”。

## 5. 不做（留 v2）
登录、历史记录（需数据库）、一键按建议自动重写、批量导入、更多平台 / 语种。

## 6. 数据结构（给 Claude Code 的契约）
生成节点输出：
```json
{ "variants": [
  { "id": 0, "lang": "en", "primaryText": "...", "headline": "...", "description": "...", "cta": "SHOP_NOW" }
]}
```
质检节点输出：
```json
{ "reviews": [
  { "variantId": 0, "status": "pass|risk", "rulesHit": ["length:headline", "banned_word:best"], "suggestions": ["..."], "fluencyScore": 88 }
]}
```

## 7. 验收标准（怎么算做完）
- [ ] 公网可访问，他人可打开
- [ ] 输入一个真实产品，返回 en + fr 各 3 组结构化文案
- [ ] 每组带质检结果（通过 / 风险 + 命中规则 + ≥1 条建议）
- [ ] API key 只在后端，前端不可见

## 8. 面试讲法（一句话锚点）
两步 LLM 编排（生成 → 质检）· 结构化输出 schema 设计 · 审核规则即配置 · badcase 迭代调参（误杀 / 漏放）
