# 出海营销内容工作台 · Overseas Marketing Content Studio

输入产品信息，自动生成 Meta（Facebook/Instagram）多语种广告文案，并对每条文案做**合规质检 + 修改建议**。一个面向出海投放运营的两步 AI 工作流工具。

## 它解决什么问题

出海投放运营要为不同市场快速产出广告文案，但面临两个痛点：一是文案要**地道、不能像机翻**，否则烧预算；二是要**合规**，绝对化词、超长字段、敏感表述都可能被广告平台拒审。本工具把"生成"和"质检"做成一条流水线，并为看不懂外文的中文运营提供译文参考。

## 核心设计：两步 LLM 工作流（生成 → 质检）

```
浏览器(React) → /api/generate(生成节点) → 卡片渲染 → /api/review(质检节点) → 徽章+建议
```

- **生成节点**只负责创意：角色设定 + 强制多角度变体 + "直接用目标语言写、不许翻译腔" + 结构化输出。
- **质检节点**只负责把关，职责单一，让两个节点各司其职、互不干扰。

> 把生成和质检拆成两个独立节点，是为了职责单一、可分别调试、可分别迭代——也让"质检"成为可信的独立关卡。

## 关键工程决策

**1. 确定性规则用代码，语义判断用 LLM（混合架构）**
质检节点不是把所有判断都丢给大模型：
- **代码层**（确定性、零成本、零延迟）：字段长度、违禁词命中——这些 100% 确定的事交给代码，准且可靠。
- **LLM 层**（判断题）：敏感/合规风险、语言地道度评分、修改建议——这些需要语义理解，才交给模型。
- **合并层**：两边结果合并出每条变体的 `pass / risk` 状态。**违禁词等确定性风险由代码强制兜底**，避免出现"命中违禁词却提示可直接使用"的不一致。

**2. 结构化输出（Structured Outputs）**
两个节点都用 strict json_schema 约束模型输出，保证返回严格匹配预定义结构，前端可直接当数据用，而非"求模型返回 JSON 再祈祷"。

**3. 规则即配置**
长度上限、违禁词表、地道度阈值都是可调配置项，运营调整审核策略无需改逻辑。阈值即"误杀/漏放"的权衡旋钮：调高更严、调低更松。

**4. 密钥安全**
LLM API key 只存在于后端 serverless 函数（环境变量），永不进入前端代码或 git 仓库。

**5. 提示词作为可版本化资产**
所有提示词与输出 schema 集中在 `docs/prompts.md` 统一管理，与代码一起版本化。

**6. 用户体验细节**
为中文运营提供每个外文字段的中文译文参考，且**复制时自动排除译文**，避免污染粘贴到广告平台的内容。

## 技术栈

| 层 | 选型 |
|---|---|
| 前端 | React + Vite |
| 后端 | Vercel Serverless Functions |
| 大模型 | OpenAI（Structured Outputs） |
| 部署 | Vercel（push 自动部署） |
| UI 设计 | Claude Design → Claude Code 落地 |

## 本地运行

```bash
npm install
# 在根目录创建 .env.local，填入：OPENAI_API_KEY=你的key
vercel dev          # 必须用 vercel dev 才能跑 /api 后端函数
```

## 目录结构

```
/src     前端（表单、卡片、复制、徽章）
/api     serverless 函数（generate.js、review.js）
/docs    PRD、提示词与 schema
CLAUDE.md  项目上下文与开发约定
```
